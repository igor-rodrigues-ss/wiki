What's System Design?

System design is the process to define architecture, components, modules and data to solve the software requirements.

Example: Think in databases

System design go beyond to chose between SQL or NoSQL database. In System Design we need to concern about:

  • Indexing strategies
  • Backup Solutions
  • Database Replications

Pillars

Based on Designing Data-Intensive Applications (Martin Kelppman) some pillars of system design are:

  • Scalability
  • Reliability
  • Maintainability

Other book like System Design Interview (Alex Xu) also quote:

  • Availabiliy

Other important aspect that isn't quoted directly like a core pillar, but is very important is:

  • Observability

So, based on my experience, the main points related System Design are:

  • Scalability
  • Reliability
  • Maintainability
  • Availabiliy
  • Observability
